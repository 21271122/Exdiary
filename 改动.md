## 当前项目改进意见：

### 一、多页面联动设计

#### 1.1 多页面相互跳转增强

当前项目，是以实验列表作为主页（被称作“仪表盘”），页面最上方一栏右侧有四个按钮，分别为收藏、时间线、分析、设置。点击其中任意按钮即可迅速跳转到相应页面。如需返回，则只需点击最上方栏左侧按钮Exdiary，即可返回仪表盘。

这种处理方式虽然简单，但是缺少一个通用的返回机制，不够灵活。我希望将五个按钮（Exdiary、收藏、时间线、分析、设置）并排整合在一起，并在所有页面增加一个返回按钮，用于回到上一个打开的页面，直到回到仪表盘。

#### 1.2 返回机制设计

系统应对页面进行编号并实时记录用户页面跳转信息，将其保存成一个列表或者其他某种形式的数据。当点击返回按钮时，依次从列表读取上一个页面编号，从而实现跳转到上一个页面的目的。

当前项目，属于“页面”分类的有：仪表盘、收藏、时间线、分析、设置、点开仪表盘中实验所跳转的实验记录、实验记录的编辑模式、点击仪表盘中新实验按钮跳转的新建实验记录页面、模板库、提取结果预览。

此外，每个页面的状态也应予以记录，用新的变量进行存储，用户对页面进行操作时，实时更新页面状态记录变量。当点击返回按钮时，被返回的页面需要读取页面状态参数，并加载正确的状态。

| 页面                     | 状态记录                                         |
|------------------------|----------------------------------------------|
| 仪表盘                    | 实验列表的滚动坐标；                                   |
| 收藏                     | 实验列表的滚动坐标；                                   |
| 时间线                    | 无；                                           |
| 分析                     | 实验列表的滚动坐标；<br/>实验的勾选情况；<br/> “你想了解什么”输入框内的文本； |
| 设置                     | 无；                                           |
| 点开仪表盘中实验所跳转的实验记录       | 无；                                           |
| 实验记录的编辑模式              | 无；                                           |
| 点击仪表盘中新实验按钮跳转的新建实验记录页面 | 文本框内文本                                       |
| 模板库                    | 无                                            |
|提取结果预览| 无                                            |

#### 1.3 布局设计

仪表盘、收藏、时间线、分析、设置五个按钮，平行排列，均匀分布在顶层栏，且顶层栏时刻保持在页面顶部，不会随着页面的滑动而离开屏幕。顶层栏左侧放一小箭头，作为返回按钮。这种布局类似手机APP风格。
